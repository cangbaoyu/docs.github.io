介绍

快速搭建属于自己的开发环境， 让你快速进入牛马状态。

Git 环境

  • 下载 Git
  • 配置好 Git 环境后，在命令行中输入 git --version 查看是否配置成功

Jenv管理多JDK版本

  • 下载 jenv
  • 在命令行中输入 jenv init 查看是否配置成功
  • 参考文章 -> Jenv-JDK管理神器.md

JDK 环境 (可选)

如果安装了Jenv，则不需要配置JDK环境系统变量，参考Jenv-JDK管理神器.md

  • 下载 JDK Downloads
  • 如果没有配置环境变量，则需要配置环境变量，打开系统环境变量，新建变量 JAVA_HOME，值为 D:\Develop\Java\jdk1.8.0_131，新建变量 PATH，值为 %JAVA_HOME%\bin
  • 配置好 JDK 环境后，在命令行中输入 java -version 查看是否配置成功
java -version

java version "1.8.0_131"
Java(TM) SE Runtime Environment (build 1.8.0_131-b11)
Java HotSpot(TM) 64-Bit Server VM (build 25.131-b11, mixed mode)

Maven 环境

  • 配置好 JDK 环境，然后下载 Maven-3.6.3
  • 解压下载好的压缩包，将解压后的文件夹重命名为 apache-maven-3.6.3，然后将文件夹放置在任意位置，如：D:\Develop\Maven\apache-maven-3.6.3
  • 配置环境变量，打开系统环境变量，新建变量 M2_HOME，值为 D:\Develop\Maven\apache-maven-3.6.3，新建变量 PATH，值为 %M2_HOME%\bin
  • 配置好 Maven 环境后，在命令行中输入 mvn -v 查看是否配置成功
mvn -v

Default locale: zh_CN, platform encoding: GBK

配置云效Maven制品仓库

Mvnd 安装

  • 配置好 JDK 环境，然后下载 Mvnd
  • 解压下载好的压缩包，将解压后的文件夹重命名为 maven-mvnd-1.0.2，然后将文件夹放置在任意位置，如：D:\Develop\Maven\maven-mvnd-1.0.2
  • 配置环境变量，打开系统环境变量，新建变量 MVND_HOME，值为 D:\Develop\Maven\maven-mvnd-1.0.2，新建变量 PATH，值为 %MVND_HOME%\bin
  • 配置好 Mvnd 环境后，在命令行中输入 mvnd -v 查看是否配置成功
# mvnd -v

  • 配置 settings.xml 路径 (可选): The location of the maven settings file. The client normally uses default settings in {@code ~/.m2/settings.xml}；可以在 conf 下的 mvnd.properties 文件自定义路径
  • 在IDEA 中配置 mvnd 配置: File -> Other Settings -> MavenHelper -> 勾选mvnd -> 右键项目 Run Maven -> clear/ install;

NVM 环境

  • 下载 NVM
  • 配置好 NVM 环境后，在命令行中输入 nvm -v 查看是否配置成功

NVM管理多Node版本

  • 下载 nvm-windows
  • 在命令行中输入 nvm -v 查看是否配置成功
PS C:\Users\cangb> nvm -v
1.1.12
PS C:\Users\cangb>
